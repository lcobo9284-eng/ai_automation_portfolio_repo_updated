# Sanitized Workflow Inventory

This inventory summarizes the workflows reviewed for this portfolio at a safe level. It is not an executable workflow export and does not include credentials, tokens, webhook URLs, prompts, private data, or production configuration.

## Murphy

The Murphy workflow included a multi tool assistant architecture with approximately forty nodes. It used AI models, Telegram and WhatsApp related interactions, HTTP requests, Supabase vector retrieval, tool workflows, message formatting, and routing logic. The workflow should not be uploaded as a raw JSON file because it contains markers related to API usage, phone IDs, and webhook configuration.

## LinkedIn Outreach System

The LinkedIn outreach workflows included orchestration agents, message generation sub workflows, company analysis, Google Sheets tracking, lead extraction, Apify related processes, PhantomBuster related execution, and Apollo search logic. Raw files should not be uploaded because they contain private lead structures, webhook IDs, authorization headers, tokens, and connected account details.

## Calendar Agent

The calendar workflows included access token retrieval, event creation, attendee coordination, meeting tools, Outlook related actions, Zoom related actions, structured output parsing, and deletion logic. Raw files should not be uploaded because they include token handling, meeting data, credentials, and private scheduling configuration.

## RAG Knowledge Ingestion

The RAG ingestion workflows included Dropbox retrieval, media transcription, document processing, text splitting, embeddings, Pinecone insertion, Airtable or Google Sheets tracking, and code nodes for data preparation. Raw files should not be uploaded because they can contain private file paths, API credentials, token markers, and internal knowledge base references.

## Voice Assistant and RAG Bot

The voice assistant workflow included Telegram message handling, voice download, transcription, vector retrieval, AI response generation, Slack or Google Sheets logging, and error handling logic. Raw files should not be uploaded because they may contain bot credentials, user message structures, knowledge base IDs, and webhook data.

## Business Report Automation

The AI Opportunity Audit, Competitive Edge Blueprint, and Brand Positioning workflows included form inputs, website fetching, HTML extraction, AI research, structured analysis, quality control, Make.com document handoff, Slack or email delivery, and Google Sheets tracking. Raw files should not be uploaded because they contain webhook data, credentials, internal prompts, document links, report payloads, and private client inputs.

## ScripCast

The ScripCast workflows included content intake, AI monologue writing, caption generation, Google Sheets tracking, HeyGen avatar video generation, Blotato upload, social platform routing, publishing actions, and final link updates. Raw files should not be uploaded because they contain API keys, social account identifiers, credential references, cached URLs, webhook IDs, and production configuration.
