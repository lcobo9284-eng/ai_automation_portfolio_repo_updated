# System Architecture Overview

This portfolio contains documentation for several automation systems. The workflows can be grouped into four main architecture patterns.

## Pattern 1: AI agent orchestration

Used for systems where an AI agent interprets a request and delegates actions to specialized tools or sub workflows.

Common components:

• User input  
• AI model  
• Memory layer  
• Structured output parser  
• Tool workflows  
• External API actions  
• Response delivery  

Applied in:

• Murphy AI Operations Assistant  
• LinkedIn Outreach Automation System  
• Calendar Automation Agent  
• RAG related assistants  

## Pattern 2: Data ingestion and knowledge retrieval

Used for workflows that transform files, documents, audio, or video into searchable knowledge.

Common components:

• File source  
• Content extraction  
• Transcription when needed  
• Text splitting  
• Embedding generation  
• Vector database insertion  
• Tracking database or spreadsheet  
• Retrieval enabled assistant  

Applied in:

• RAG Knowledge Ingestion System  
• Voice Assistant and RAG Bot  
• Call Paul knowledge update flows  

## Pattern 3: Business audit report generation

Used for workflows that collect company inputs, analyze public information, and generate strategic business reports.

Common components:

• Form input  
• Website fetch  
• HTML extraction  
• AI research  
• Structured analysis  
• Quality control  
• Document generation  
• Email or Slack delivery  

Applied in:

• AI Opportunity Audit Generator  
• Competitive Edge Blueprint  
• Brand Positioning Audit and GTM Blueprint  

## Pattern 4: Outreach and CRM automation

Used for workflows that support prospecting, lead enrichment, messaging, and campaign tracking.

Common components:

• Audience criteria  
• Prospect search  
• Lead data extraction  
• AI analysis  
• Personalized messaging  
• Follow up sequence  
• Google Sheets or CRM tracking  

Applied in:

• LinkedIn Outreach Automation System  
• Campaign follow up workflows

## Pattern 5: AI content generation and social publishing

Used for workflows that convert content ideas into generated scripts, captions, avatar videos, published posts, and tracking records.

Common components:

• Form input  
• AI script generation  
• Caption generation  
• Content tracking database  
• Avatar video generation  
• Wait and status checking logic  
• Media upload service  
• Social publishing API  
• Platform based routing  
• Final output tracking  

Applied in:

• ScripCast AI Video Content Automation System
