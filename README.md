## My working approach

I start by understanding the real process before building the automation. I look for manual work, duplicated steps, disconnected tools, unstable handoffs, unclear data structures, and places where teams lose visibility. Once the problem is clear, I design a workflow that connects the right tools, defines clean inputs and outputs, includes error handling, and remains understandable for future maintenance.

My focus is not only making a workflow run once. I care about whether the system is readable, traceable, scalable, and useful for the team that depends on it. I prefer clear architecture, structured data, practical AI usage, reliable integrations, and documentation that helps another technical person understand the system without guessing.
