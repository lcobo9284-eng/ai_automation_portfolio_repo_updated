# Confidentiality Notice

This repository is a professional portfolio. It is designed to explain automation experience without exposing private implementation details from previous work.

The portfolio includes sanitized project descriptions, architecture summaries, workflow logic, tool references, documentation templates, and safe explanations of technical decisions. It does not include proprietary code, raw n8n workflow exports, credentials, API keys, access tokens, webhook URLs, database identifiers, account IDs, internal prompts, lead lists, client data, private documents, raw reports, or confidential business information.

This distinction matters because automation workflows often store sensitive details inside nodes, headers, credentials, pinned data, prompts, test payloads, and external service configurations. Publishing a raw workflow export can expose private systems even when the visible project seems harmless.

The publication standard for this repository is simple. Each project should be understandable to a recruiter or technical reviewer, but it should not allow anyone to access, recreate, or identify private client systems. Any example shared publicly should be anonymized, recreated with fictitious data, or written as a high level case study.
