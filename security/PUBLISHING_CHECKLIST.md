# Publishing Checklist

Use this checklist before uploading any automation project to GitHub.

## Security review

• Remove all API keys  
• Remove all bearer tokens  
• Remove all client secrets  
• Remove all webhook URLs  
• Remove all account IDs that identify private systems  
• Remove all credential blocks from workflow exports  
• Remove all pinned data that includes real names, emails, phone numbers, company records, or client information  
• Remove internal prompts if they belong to a previous employer or client  
• Remove private Google Sheet, Airtable, Notion, Dropbox, Slack, Telegram, WhatsApp, or CRM identifiers  
• Replace real company data with fictitious examples  
• Confirm that no confidential business strategy is included  

## Portfolio review

• Describe the business problem without exposing the client  
• Explain your role clearly  
• Mention tools used at a high level  
• Focus on workflow logic and impact  
• Avoid uploading raw production workflows  
• Use diagrams and summaries instead of sensitive code  
• Add a confidentiality note to every case study when needed  

## Recommended public format

• README overview  
• Anonymized case studies  
• Architecture summaries  
• Tool list  
• Methodology notes  
• Recreated demo workflows only when safe  
• Sample Python scripts using fictitious data only
